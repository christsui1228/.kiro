# 任务管理系统规则

## 任务管理命令格式

### 支持的命令：
1. **查看任务列表** - `查看任务` 或 `任务列表`
2. **添加新任务** - `添加任务: [任务描述]`
3. **修改任务** - `修改任务 [任务ID]: [新描述]`
4. **删除任务** - `删除任务 [任务ID]`
5. **执行任务** - `执行任务 [任务ID]`
6. **标记完成** - `完成任务 [任务ID]`

## 任务文件规范

### 文件位置：
- 任务列表文件：`todolist.md`（项目根目录）

### 任务格式：
```markdown
# 项目任务列表

## 待办任务
- [ ] [ID-001] 任务描述 - 创建时间：YYYY-MM-DD
- [ ] [ID-002] 任务描述 - 创建时间：YYYY-MM-DD

## 进行中任务
- [⏳] [ID-003] 任务描述 - 开始时间：YYYY-MM-DD

## 已完成任务
- [x] [ID-004] 任务描述 - 完成时间：YYYY-MM-DD
```

## 执行规则

### 任务ID生成：
- 格式：ID-XXX（XXX为三位数字，从001开始）
- 自动递增，不重复使用

### 任务状态：
- `[ ]` - 待办
- `[⏳]` - 进行中
- `[x]` - 已完成

### 操作权限：
- 只能修改和删除未完成的任务（待办和进行中）
- 已完成任务不可修改，只能查看

### 执行任务时：
1. 先将任务状态改为进行中 `[⏳]`
2. 询问用户确认执行细节
3. 按照开发流程策略执行
4. 完成后标记为已完成 `[x]`

## 命令处理流程：
1. 识别用户的任务管理命令
2. 读取当前 todolist.md 文件
3. 执行相应操作
4. 更新文件内容
5. 向用户确认操作结果
